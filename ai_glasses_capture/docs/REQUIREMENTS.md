# AI 眼镜数据采集 App — 需求文档

> 版本：v0.1  
> 日期：2026-06-30  
> 状态：需求梳理 / 原型阶段

---

## 1. 项目背景

用户拥有一款 AI 眼镜，眼镜通过 WiFi 与手机通信，将采集到的**视频**和**音频**数据传送到手机侧。本 App 的核心目标是：

1. 连接眼镜所在的 WiFi 网络（或让手机加入眼镜热点）
2. 以网络抓包或直连流的方式获取音视频数据
3. 解码后保存到手机本地，供后续分析或回放

这是一个**以数据采集为主**的工具型 App，界面需要美观、易用，但功能逻辑应保持简单清晰。

---

## 2. 目标用户与使用场景

| 角色 | 场景 |
|------|------|
| 个人开发者 / 测试人员 | 调试眼镜固件、验证传输协议 |
| 数据采集员 | 按被试信息归档采集片段 |
| 后续算法同学 | 读取本地存储的音视频做训练或评测 |

**典型流程：**

```
填写个人信息 → 选择/连接设备 → 开始采集 → 实时预览（可选）
→ 停止采集 → 自动解码存储 → 在记录列表中查看/导出
```

## 3. 功能需求

### 3.1 用户信息（P0）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 被试或操作者姓名 |
| 编号 | 文本 | 否 | 实验编号 / 工号 |
| 备注 | 多行文本 | 否 | 场景说明、佩戴位置等 |
| 采集日期 | 日期 | 自动 | 默认当天，可选手动修改 |

信息用于文件命名与元数据归档，例如：`张三_20260630_143022.mp4`。

### 3.2 设备连接（P0）

| 功能 | 说明 |
|------|------|
| 设备下拉选择 | 预设常见 AI 眼镜型号（可扩展），如「通用协议 / XX 眼镜 A / XX 眼镜 B」 |
| WiFi 状态展示 | 当前 SSID、IP 地址、连接状态（未连接 / 连接中 / 已连接） |
| 一键连接 | 引导用户连接眼镜热点，或检测是否已在同一局域网 |
| 手动 IP / 端口 | 高级选项：填写眼镜推流地址（如 `192.168.4.1:8554`） |
| 连接测试 | Ping / HTTP 探测，确认设备可达 |

### 3.3 数据采集（P0）

| 功能 | 说明 |
|------|------|
| 开始采集 | 建立与眼镜的数据通道，开始接收音视频 |
| 停止采集 | 结束当前会话，触发解码与落盘 |
| 采集时长显示 | 实时显示已采集秒数 |
| 实时预览 | 视频画面预览（若协议支持）；音频电平指示 |
| 抓包模式开关 | 高级：启用 VPN 本地抓包（Android）；直连流模式（优先） |

### 3.4 存储与记录（P0）

| 功能 | 说明 |
|------|------|
| 本地存储 | 视频 `.mp4` / 音频 `.aac` 或 `.wav`，存于 App 私有目录 |
| 元数据 | JSON 或 SQLite 记录：姓名、设备、时间、文件路径、时长、大小 |
| 记录列表 | 按时间倒序展示历史采集 |
| 播放 / 分享 | 系统播放器打开；导出到文件 App 或 AirDrop / 微信等 |
| 删除 | 单条删除，带确认弹窗 |

### 3.5 设置（P1）

| 功能 | 说明 |
|------|------|
| 存储路径说明 | 展示占用空间、一键清理缓存 |
| 视频质量 | 原画 / 压缩（后期） |
| 深色模式 | 跟随系统 / 手动切换 |
| 关于 | 版本号、协议说明、隐私政策链接 |

## 4. 界面需求

### 4.1 设计风格

- **审美**：接近主流消费级 App（参考 iOS Human Interface / Material Design 3）
- **色调**：深色底 + 蓝紫渐变强调色，或简洁白底卡片式
- **组件**：圆角卡片、底部 Tab 导航、浮动主按钮、状态徽章
- **反馈**：连接中 Loading、成功 Toast、错误 Alert

### 4.2 页面结构（4 Tab + 1 采集页）

```
┌─────────────────────────────────┐
│  首页（采集）  │  设备  │  记录  │  我的  │
└─────────────────────────────────┘
```

| 页面 | 核心元素 |
|------|----------|
| **首页 / 采集** | 大预览区、开始/停止按钮、时长、连接状态条 |
| **设备** | 设备下拉框、WiFi 信息、IP 输入、连接/测试按钮 |
| **记录** | 列表卡片（缩略图、姓名、时长、日期）、搜索 |
| **我的** | 姓名等表单、设置入口、存储用量 |
| **采集进行中** | 全屏预览 + 停止按钮（可选独立页） |

### 4.3 交互细节

- 未连接设备时，「开始采集」置灰并提示先连接
- 采集中切换 Tab 不中断采集（后台继续）
- 表单失焦自动保存草稿
- 下拉框支持「添加自定义设备名」

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 1080p@30fps 采集时 UI 不卡顿；解码异步进行 |
| 稳定性 | 断 WiFi 自动重连（3 次重试）；异常不丢已缓冲数据 |
| 兼容性 | Android 10+；iOS 15+ |
| 隐私 | 数据默认仅存本地，不上传云端 |
| 扩展性 | 协议层抽象，便于接入不同眼镜型号 |

## 6. 分期交付

### Phase 1 — MVP（4~6 周）

- [ ] 用户信息表单 + 本地持久化
- [ ] WiFi 连接状态检测
- [ ] 直连流拉取（HTTP / RTSP / TCP，视眼镜协议而定）
- [ ] FFmpeg 或系统解码器落盘 MP4
- [ ] 记录列表 + 播放

### Phase 2 — 抓包与适配（4 周）

- [ ] Android VPN 抓包模块
- [ ] 协议解析器（从 PCAP 提取 H.264/AAC）
- [ ] 多设备配置模板

### Phase 3 — 体验与上架（4 周）

- [ ] UI 动画与细节打磨
- [ ] 崩溃监控、日志导出
- [ ] 隐私政策、应用商店素材
- [ ] 小米 / 华为 / App Store 提交

## 7. 风险与待确认项

| 项 | 说明 | 建议 |
|----|------|------|
| 眼镜传输协议 | RTSP？自定义 TCP？UDP 组播？ | **开发前用 Wireshark 抓一次包确认** |
| iOS 抓包限制 | 系统不允许普通 App 做全局抓包 | iOS 优先走**直连 API/流**，抓包仅 Android |
| 厂商 SDK | 是否有官方 Android/iOS SDK | 有 SDK 则优先 SDK，抓包作兜底 |
| 编码格式 | H.264 / H.265 / MJPEG | 决定解码器选型 |
| 音频格式 | PCM / AAC / Opus | 同上 |

---

## 8. 验收标准（MVP）

1. 手机连接眼镜 WiFi 后，App 显示「已连接」
2. 填写姓名，选择设备，点击「开始采集」
3. 采集 30 秒以上，停止后本地生成可播放的 MP4
4. 记录页可看到该条记录，含姓名、时长、时间
5. 重启 App 后记录仍在
